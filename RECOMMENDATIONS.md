Редактор монтажных схем отопительного оборудования MyHeat.

- **Backend:** Laravel 13, PHP 8.3–8.4, MySQL
- **Frontend:** React 19, Vite 8, `react-konva` (canvas), экспорт PDF через `jspdf`

## 🔴 Безопасность

### 1. Нет аутентификации и авторизации
Все эндпоинты в `routes/web.php` публичны, `bootstrap/app.php` имеет пустой `withMiddleware()`.
Любой пользователь может читать, изменять и удалять любые схемы, включая массовое удаление
(`DELETE /api/schemes` с массивом id). Страницы `/admin`, `/scheme`, `/selection` открыты.
Модель `User` и таблица `users` существуют, но не используются — `user_id` захардкожен как `1`.

**План:**
- [ ] Ввести `auth` middleware (Laravel Breeze/Sanctum или сессионная авторизация).
- [ ] Привязывать схему к реальному `Auth::id()` вместо `user_id => 1`.
- [ ] Добавить Policy на `Scheme` (просмотр/редактирование/удаление только владельцем/ролью).
- [ ] Защитить `/admin` отдельной ролью.

### 2. Открытый прокси с пробросом cookie
`/api/proxy/integration` и `/api/integration` пересылают тело запроса **и заголовок `Cookie`**
клиента на `https://mhtest.ru` с подделкой `Origin`/`Referer`. Риск утечки сессий и SSRF-подобного
шлюза без валидации.

**План:**
- [ ] Убрать проброс клиентского `Cookie`; использовать серверный токен интеграции из `.env`.
- [ ] Валидировать/ограничивать тело запроса (allow-list полей и `action`).
- [ ] Вынести интеграцию в отдельный сервис-класс с таймаутами и обработкой ошибок.

### 3. Дефолтный пароль суперпользователя
`superuser123` захардкожен в `DatabaseSeeder` и продублирован в `docker-compose.yml`.

**План:**
- [ ] Убрать пароль из `docker-compose.yml` и дефолт из сидера.
- [ ] Требовать `SUPERUSER_PASSWORD` из окружения, падать при отсутствии в проде.

### 4–6. Прочее
- [ ] Добавить `throttle` на пишущие/удаляющие эндпоинты.
- [ ] Ограничить размер и глубину `incoming_scheme` (правило валидации + лимит на колонку/размер запроса).
- [ ] Убрать `APP_DEBUG=true` из прод-путей; сузить права вместо `chmod -R 777` в `entrypoint.sh`.

## 🟠 Архитектура и поддерживаемость

### 7. Монолитный `spa.jsx` (16 000+ строк)
Один файл невозможно ревьюить, он тормозит IDE и провоцирует конфликты и регрессии.

**План (постепенно, без переписывания «с нуля»):**
- [ ] Выделить слои: `rendering/` (компоненты Konva), `state/` (стор/редьюсеры), `interactions/` (drag, hover, слот-меню).
- [ ] Вынести крупные подсистемы в отдельные компоненты/хуки.
- [ ] Ввести стор (Zustand/Redux Toolkit) вместо разросшегося локального состояния.
- [ ] То же для `selection.jsx` (4400 строк).

### 8. Бэкенд-логика в замыканиях роутов
**План:**
- [ ] Создать `SchemeController` (resource) и `IntegrationController`.
- [ ] Вынести валидацию в `FormRequest`.
- [ ] Вынести бизнес-логику в Actions/Services; убрать дублирование JSON-ответов (API Resources).

### 9. Хак с `view.compiled`
- [ ] Настроить компиляцию вью через `.env`/конфиг хостинга вместо переопределения в `AppServiceProvider::boot()`.

4. **Этап 3 (качество):** тесты + запуск их в CI; разделение dev/prod окружений, staging, откат.
5. **Этап 4 (гигиена):** ассеты, конвенция коммитов.
